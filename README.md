# Radar de Proximidad con Efecto de Persistencia (ESP32)

## 1. Idea General del Proyecto
Este proyecto consiste en el diseño y construcción de un radar de barrido sonar interactivo controlado por un microcontrolador ESP32. El sistema simula el comportamiento de un radar militar o marítimo real, donde un sensor ultrasónico gira constantemente de forma horizontal buscando obstáculos en el entorno. Cuando el sensor detecta un objeto, calcula su posición exacta mediante variables de ángulo y distancia para enviarla a una pantalla OLED, espacio en el cual se dibuja un mapa bidimensional en tiempo real. El sistema destaca por su fluidez visual y por su capacidad de mantener un registro gráfico dinámico de los objetos detectados mientras el sensor continúa su trayectoria de exploración.

---

## 2. Componentes Usados y su Función
El núcleo del sistema está compuesto por cinco elementos de hardware que interactúan de forma coordinada. En primer lugar, el microcontrolador ESP32 actúa como el cerebro del robot, encargándose de procesar la trigonometría, controlar los tiempos del motor, medir los pulsos del ultrasonido, gestionar la memoria y renderizar los gráficos en la pantalla. Como ojos del radar se utiliza el sensor ultrasónico HC-SR04, el cual emite ráfagas de alta frecuencia y mide el tiempo que tarda el eco en regresar tras chocar con un objeto, permitiendo calcular la distancia exacta en centímetros. El cuello mecánico que permite rotar este sensor con precisión milimétrica es el motor paso a paso 28BYJ-48, un dispositivo que divide una vuelta completa en 4096 micro-pasos gracias a su caja reductora interna. Para controlar este motor de forma segura se incluye el módulo controlador ULN2003, el cual actúa como un puente o músculo de potencia; dado que el motor requiere cerca de 300 mA para girar y la ESP32 solo entrega señales lógicas delicadas, este chip toma la energía pesada directamente de la línea del cable USB sin sobrecargar el procesador. Finalmente, la pantalla OLED SSD1306 sirve como el monitor visual del sistema, mostrando una interfaz gráfica PPI (Plan Position Indicator) con una cuadrícula de radar, una línea de barrido síncrona y los puntos que representan los obstáculos.

---

## 3. ¿Qué Hace el Código? (Arquitectura del Software)
El firmware del proyecto está estructurado de forma inteligente para optimizar el rendimiento de la ESP32 a través de varias etapas consecutivas que gestionan la memoria, el movimiento y los gráficos. Al encenderse el dispositivo, el programa ejecuta la fase de inicialización configurando los pines de entrada y salida, y recurre a la librería Preferences para leer la memoria Flash permanente del chip; esto permite recuperar el último paso en el que se apagó el motor y resolver el problema de que los motores paso a paso sean mecánicamente "ciegos", evitando así que el robot pierda su orientación espacial o su punto cero de partida.

Una vez superada la configuración inicial, el sistema entra en un bucle infinito donde el motor realiza un viaje continuo de ida y vuelta entre el paso 0 y el 4096. Para garantizar un desplazamiento fluido, el software implementa un filtro matemático que le permite avanzar continuamente y detenerse a escanear únicamente cada 40 pasos. En cada una de estas paradas de control, la ESP32 realiza una conversión angular mediante la función map para traducir los pasos del motor a grados sexagesimales de un círculo, dispara el sensor ultrasónico para calcular la distancia exacta hacia el obstáculo en centímetros y aplica un filtro de control de calidad que descarta lecturas falsas, almacenando solo los datos válidos en la matriz de memoria del mapa.

Finalmente, toda esta información se procesa en la etapa de renderizado, la cual utiliza funciones trigonométricas de seno y coseno para proyectar de forma matemática la posición de los objetos sobre el plano circular de la pantalla OLED. Para optimizar la experiencia visual y evitar que los puntos parpadeen o desaparezcan inmediatamente cuando la línea de barrido avanza, el código añade un algoritmo de persistencia de radar que otorga un "tiempo de vida" a cada detección. De esta manera, cada obstáculo descubierto pierde una unidad de vida de forma matemática con cada paso del radar, lo que visualmente se traduce en una elegante estela brillante que se desvanece suavemente en la pantalla a lo largo de aproximadamente dos segundos, imitando a la perfección el comportamiento de las pantallas de barrido de los radares profesionales.

## 4. Lógica de Escaneo de 360° y Datos Utilizados
El secreto detrás del barrido completo o viaje de vaivén que simula el radar radica en cómo la función loop gestiona las llamadas de movimiento pasándole diferentes coordenadas de destino como si fueran metas métricas. En la primera línea del ciclo principal, el código ordena al motor avanzar hacia la variable con el valor fijo de 4096 pasos, obligando al eje a girar hacia la derecha completando una vuelta físicamente perfecta mientras el sensor ultrasónico toma muestras espaciadas del entorno. Una vez que el motor alcanza esa meta final y hace una breve pausa de medio segundo para que el usuario aprecie el mapa dibujado, el ciclo ejecuta la siguiente instrucción mandando al motor de regreso a la posición de paso cero. Como el software calcula automáticamente la diferencia de distancia, el motor invierte su sentido de giro hacia la izquierda, realizando un escaneo de regreso por los mismos puntos hasta alinearse nuevamente con el origen físico del robot.

Para que la pantalla OLED pueda interpretar mecánicamente este viaje de miles de pasos y plasmarlo en un círculo visual armónico, el código recurre a la función map dentro de los bloques de escaneo para traducir la escala de pasos del motor (0 a 4096) a la escala de grados geométricos tradicionales (0° a 360°). De este modo, cuando el motor se encuentra físicamente a la mitad de su trayectoria en el paso 2048, el algoritmo calcula instantáneamente que el sensor está apuntando exactamente a los 180°. Con este dato angular purificado, la función de dibujo aplica fórmulas de trigonometría plana para calcular las coordenadas espaciales en los píxeles de la pantalla, logrando proyectar una línea de barrido que gira en perfecta sincronía con el hardware y pintando los obstáculos en la dirección real en la que existen en la habitación.
